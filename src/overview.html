Documentation for GSMA SDK

This SDK provides an all-in-one package of client library functions for:

  • Operator API Discovery - allowing an application to determine the operator and available API services for the user

  • Operator/API Logo - allowing an application to obtain 'official' image resources that can be used in the application user interface

  • Mobile Connect API Services - allowing an application to use authentication services linked with a mobile phone account

Operator API Discovery

Use the 'discovery' API to determine the operator and available API services for the mobile user. This API works with a network based discovery service together using various mechanisms to identify the operator.

Once this is done the application is notified of the available API services and associated 'meta data' for the operator.

See {@link com.gsma.android.oneapi.discovery.DiscoveryProvider} for details on the main interface for the discovery service

Operator/API Logo

Use the 'logo' API to obtain 'official' logos/ images relevant to the operator and API combination. This is related to the discovery API and in case the operator is known allows the application to see if operator specific logos are available.

The 'logo' API works with a network based logo store service and handles the retrieval of image resources as a convenience to the application.

See {@link com.gsma.android.oneapi.logo.LogoProvider} for details on the main interface for the discovery service

Mobile Connect

The Mobile Connect service is based on OAuth2/ OpenID Connect API services, but specialised for use with mobile networks providing mobile authentication services. Applications can be sure that there is a real end user accessing the service, with access validated through the use of the mobile device.

Mobile Connect provides a common API service across operators independent of the authentication methods preferred by the home network operator.

The Mobile Connect SDK provides a convenient interface allowing flexibility over the use of Mobile Connect and the OAuth2/ OpenID Connect API services on which it is based.

See the following main interfaces for Mobile Connect:

  • {@link com.gsma.android.mobileconnect.authorization.Authorization} - sign in to the application using the user's Mobile Connect account as a secure alternative to offering username/ password based registration or login. See {@link com.gsma.android.mobileconnect.authorization.Authorization} for the main interface for application authorization.

  • {@link com.gsma.android.mobileconnect.token.Token} - once the user is authorized the application receives a one time use code which is exchanged for a reusable access token. See {@link com.gsma.android.mobileconnect.token.Token} for the main interface for exchanging an authorization code for a token.

  • {@link com.gsma.android.mobileconnect.userinfo.Userinfo} - if supported by the Mobile Network and subject to user consent allows the application to receive 'user information' fields such as email address, physical address etc using a previously obtained access token. See {@link com.gsma.android.mobileconnect.userinfo.Userinfo} for the main interface for obtaining 'user information' based on an access token.

Dependencies

Operation of this SDK is dependent on the following external libraries

  • Google Android Support Library - version 4 (or later) which can be obtained from http://developer.android.com/tools/support-library/index.html

  • The Jackson JSON library - version 1.9.3 which can be obtained from https://github.com/FasterXML/jackson
